## 一、背景与目标

当前 WBS 项目管理系统已经具备任务/项目管理、甘特图、周报、文档、加班、延期通知、5 角色权限等基础能力。本次迭代围绕"**让项目数据更真实、运营数据更可见、组织与权限更灵活**"三个方向，新增四项能力：

| 序号 | 功能 | 业务价值 |
|---|---|---|
| 1 | **项目/任务 计划日程 vs 实际日程** | 终结"计划靠拍脑袋、实际靠回忆"的现状；延期、报表、追溯都拿到一手数据 |
| 2 | **菜单点击率统计** | 量化用户行为，发现"没人用"和"卡顿"的功能，为后续裁剪/优化提供依据 |
| 3 | **组织结构展示** | 一图看清公司/部门/小组，看到每节点下的人数/项目数/任务数/延期数 |
| 4 | **角色管理 + 权限码编辑** | 管理员不再依赖开发改权限，业务侧可自助调整各角色能做什么 |

---

## 二、范围与非范围（业务口径）

### ✅ 本期做

1. **实际日程** — 任务/项目都加"实际开始日 / 实际结束日"。**无需用户手动填**，由状态变更自动回填（任务进入"进行中"自动记开始、任务"完成"自动记结束；项目按子任务聚合）。
2. **菜单点击率** — 10 个侧边栏菜单项的 PV/UV，按角色/按日期统计；admin 可见独立"使用统计"页面。
3. **组织结构** — 复用 HR/MDM 现有数据，**只读 + 可视化**（树形 + 节点详情 + Excel 导出）。
4. **角色管理** — **保留现有 5 固定角色**（管理员/部门项目负责人/项目负责人/项目人员/观察者），admin 在 UI 上勾选各角色能拥有的权限码，立即生效；写入有审计日志。

### ❌ 本期不做（明确划清边界）

- **自定义角色**（如"财务审批人"）— 6/30 截止前压力大，**保持 5 角色固定**
- **组织节点增删改** — 现有 MDM/HR 同步链路已稳定，**只读即可**
- **dhtmlx-gantt "实际线"绘制** — 甘特图开源版不支持，**仅在 tooltip 展示**
- **全量按钮级埋点** — **仅做侧边栏 10 项**，控制工作量
- **Redis / 集群 / 压测** — 单实例部署，**不引入新基础设施**

---

## 三、关键决策（已与业务方确认）

| 决策点 | 选择 | 理由 |
|---|---|---|
| 实际日程怎么来 | 状态变更自动回填 | 用户最省心、不漏填、零培训成本 |
| 组织结构深度 | 只读 + 可视化 | 数据已从 HR/MDM 自动同步，无需双写 |
| 是否支持自定义角色 | **保持 5 固定角色** | 6/30 截止前实现自定义角色风险高，权限码层面已能覆盖大部分差异 |
| 菜单点击率范围 | 仅侧边栏 10 项 | 满足"知道哪个菜单热门"需求，避开全量埋点工作量 |

## 四、四个功能要点

### 1. 项目/任务 计划日程 vs 实际日程

- **展示侧**：
  - 项目详情卡：原"计划 2026-06-01 ~ 2026-06-30" 行下面增加一行只读"**实际 2026-06-02 ~ 2026-06-28**"，无值显示"—"
  - 任务详情卡：同样双行展示
  - 甘特图 tooltip：把"实际"数据流补上（文案已有，缺数据）
- **数据侧**：
  - 项目/任务表各加 2 个日期字段（`actualStartDate` / `actualEndDate`）
  - 任务状态进入"进行中" → 自动写开始日；任务"完成" → 自动写结束日
  - 项目按子任务聚合（首个任务启动 → 项目开始日；全部任务完成 → 项目结束日）
  - 状态回退（如"完成 → 进行中"）保留已记录的实际结束日，**不抹除审计**
- **报表侧**：项目延期报表、周报自动汇总"计划 vs 实际"差异，**所有现有报表零改动**即可受益

### 2. 菜单点击率统计

- **采集侧**：用户每次点击侧边栏菜单项，前端异步写一条日志（userId + menuCode + clickTime + 角色快照）
- **查看侧**：admin 进入新页面"**使用统计**"，看到
  - 顶部 4 个数字卡（7 天总 PV / 7 天 UV / 30 天 PV / 最热门菜单）
  - 柱图：10 个菜单 PV 排行
  - 折线图：选中某菜单后 30 天趋势
  - 饼图：按角色分布
- **admin 菜单位置**：侧边栏最底部"管理员专区"

### 3. 组织结构

- **页面布局**：
  - 左侧 30%：组织树（公司 → 部门 → 小组），可展开/折叠
  - 右侧 70%：选中节点的 4 个数字卡（用户数 / 项目数 / 任务数 / 延期任务数）+ 用户表 + 项目表
  - 顶部"导出 Excel"按钮（导出该节点及所有子孙的用户/项目清单）
- **数据源**：完全来自 HR/MDM 现有数据，**不复制、不新增表**
- **admin 菜单位置**：侧边栏"管理员专区"第二项

### 4. 角色管理 + 权限码编辑

- **入口**：admin 进 Settings → 新 Tab "**角色权限**"（与个人资料、通知、显示、安全、HR 同步并列）
- **操作**：
  - 顶部下拉选角色（5 个固定值）
  - 中部按"菜单 / 操作 / 数据范围"分组列出所有权限码，每行一个勾选框
  - 底部"保存"按钮（弹确认对话框，提示"将立即对所有用户生效"）
  - 旁边"查看历史"按钮，看最近 20 次修改记录
- **生效机制**：
  - 写入 `sys_role_permission` 关联表
  - 自动失效权限缓存
  - 下次请求立即按新权限执行（无需重新登录）
  - 所有写入记审计日志（userId、角色、变更前/后的权限码列表、时间）

## 六、风险与缓解

| 风险 | 影响 | 缓解 |
|---|---|---|
| 6/30 截止时间紧，4 项功能都是前后端联动 | 延期 | 周末各安排 1 天缓冲；M1（6/18）若 actualDates 未通过则立即砍甘特 tooltip 收口 |
| 修改角色权限立即影响所有在线用户 | 操作失误可能锁死全公司账号 | 保存前弹二次确认对话框；写入审计日志便于回滚；admin 自身不受影响 |
| 菜单点击日志长期累积导致表膨胀 | 半年后查询慢 | 加 90 天归档策略（admin 手动启用），本期不写自动 job |
| 状态变更自动回填可能与历史任务不一致 | 老任务没有 actualDates | 现状保留 null，UI 显示"—"；后续可选"补录"端点（本期不做） |
| dhtmlx-gantt 开源版不支持"实际线" | 业务方可能要求画线 | **本期只做 tooltip 展示**；如要画线需评估 dhtmlx-gantt pro license 成本 |

## 七、验收标准（业务方视角）

每项功能由业务方 5 分钟手测即可验收：

| 功能 | 验收动作 | 期望结果 |
|---|---|---|
| **实际日程** | 建任务 → 改状态 in-progress → 改 done | 任务详情卡"实际"行从"—"变成今天日期 |
| **菜单点击率** | 点 5 次"项目管理"、3 次"周报" → admin 进"使用统计" | 柱图显示"项目管理 PV=5"、"周报 PV=3" |
| **组织结构** | admin 进"组织架构" → 展开 2700 公司 → 点某部门 | 右侧 4 个数字卡显示该部门真实人数/项目/任务/延期数 |
| **角色权限** | admin 取消勾选 member 的"项目：新建" → 用 member 账号登录 | "新建项目"按钮消失；查审计日志有 1 条记录 |
