# Excel 导出功能测试指南

## 测试准备

### 1. 启动应用
```bash
# 进入前端目录
cd frontend

# 启动开发服务器
npm run dev
```

### 2. 访问 Reports 页面
1. 登录系统
2. 在侧边栏找到"报表"或"Reports"菜单
3. 点击进入 Reports 页面

## 测试场景

### 主要场景：导出综合报表

**步骤：**
1. 确保系统中至少有一个项目
2. 在 Reports 页面底部找到"导出"卡片
3. 点击"导出综合报表"按钮
4. 等待文件下载

**预期结果：**
- ✅ 下载名为 `综合报表_YYYYMMDD_HHMM.xlsx` 的文件
- ✅ 文件包含 4 个工作表：
  1. **项目总览** - 10列，包含所有项目的详细信息
  2. **任务明细** - 10列，包含所有叶子任务的详细信息
  3. **成员列表** - 7列，包含所有成员的信息
  4. **统计汇总** - 关键指标汇总
- ✅ 所有工作表数据正确
- ✅ 关联数据正确（如任务关联到项目、负责人姓名正确）
- ✅ 计算字段正确（如进度百分比、延期天数）
- ✅ 中文显示正常，无乱码
- ✅ 日期格式为 YYYY-MM-DD
- ✅ 状态和优先级显示为中文（如"进行中"、"高"等）

## 性能基准

### 响应时间
- 小数据量（< 10个项目）：导出 < 1秒
- 中数据量（10-50个项目）：导出 < 3秒
- 大数据量（50-100个项目）：导出 < 10秒

### 文件大小
- 小数据量：< 100 KB
- 中数据量：100 KB - 1 MB
- 大数据量：1 MB - 10 MB

## 问题排查

### 问题 1：点击按钮无反应
**可能原因：**
- 数据未加载完成
- JavaScript 错误

**解决方案：**
1. 打开浏览器开发者工具（F12）
2. 查看 Console 标签页是否有错误
3. 确认项目、任务、用户数据已加载

### 问题 2：文件无法打开
**可能原因：**
- xlsx 库版本不兼容
- 文件损坏

**解决方案：**
```bash
# 重新安装 xlsx
cd frontend
npm uninstall xlsx
npm install xlsx@0.18.5
```

### 问题 3：中文乱码
**可能原因：**
- 编码问题
- xlsx 库版本问题

**解决方案：**
- 确保使用 xlsx 0.18.5 版本
- 检查浏览器下载设置

### 问题 4：数据不完整
**可能原因：**
- 数据过滤逻辑错误
- 关联数据缺失

**解决方案：**
- 检查浏览器控制台是否有错误
- 确认数据已正确加载到 store 中
- 验证用户 ID、项目 ID 等关联数据是否存在

### 问题 5：只显示部分数据
**可能原因：**
- Excel 版本限制
- 数据格式问题

**解决方案：**
- 使用最新版本的 Excel（2007+）
- 检查数据中是否有特殊字符
- 尝试重新导出

## 需要帮助？
如果遇到问题，请按以下顺序检查：
1. 浏览器控制台是否有错误信息（F12 -> Console）
2. 网络请求是否正常（F12 -> Network）
3. 数据是否正确加载到 Pinia store 中（F12 -> Vue DevTools）
4. xlsx 库版本是否为 0.18.5（`npm list xlsx`）
5. Excel 版本是否支持 .xlsx 格式（Excel 2007+）
