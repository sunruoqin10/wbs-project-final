# WBS项目报表导出功能开发方案

## 一、功能概述

### 1.1 目标
为WBS项目管理系统提供完善的报表导出功能，支持多种导出类型，满足不同场景下的数据分析和汇报需求。

### 1.2 导出类型

| 导出类型 | 描述 | 适用场景 |
|---------|------|---------|
| 项目Excel | 导出项目列表基本信息 | 项目概览、快速查看 |
| **周报导出** | 本周项目进度和团队绩效 | 周会汇报、进度跟踪 |

### 2.3 核心逻辑

#### 任务筛选逻辑
```typescript
// 本周任务判断条件
function getTasksInWeek(tasks, weekStart, weekEnd) {
  return tasks.filter(task => {
    const taskStart = task.startDate || task.createdAt;
    const taskEnd = task.endDate || now;
    // 任务时间范围与本周有交集
    return taskStart <= weekEnd && taskEnd >= weekStart;
  });
}

// 本周完成任务判断
function getCompletedTasksInWeek(tasks, weekStart, weekEnd) {
  return tasks.filter(task => {
    return task.status === 'done' && 
           task.completedAt >= weekStart && 
           task.completedAt <= weekEnd;
  });
}

// 延期任务判断
function getDelayedTasksInWeek(tasks, weekStart, weekEnd) {
  return tasks.filter(task => {
    return task.endDate < weekEnd && task.status !== 'done';
  });
}
```

## 六、注意事项

1. **日期计算**：使用 dayjs 的 week() 方法计算周数，注意周一为一周的开始
2. **任务统计**：只统计叶子任务，避免父子任务重复计数
3. **性能优化**：大量数据时考虑分批处理
4. **错误处理**：添加完善的错误提示和异常捕获
5. **文件命名**：包含时间戳，避免文件名冲突
